# 更新日志

## v2.3.1 - 2026-07-22

- 优化启动和设置切换速度，新增后台预加载以加快模式切换。
- 账号信息读取失败时保留上次的邮箱和类型，并完整换行显示错误。

## v2.3.0 - 2026-07-19

- 新增账号 Token 使用总量展示。
- 新增独立的 Token 用量模式。
  - 读取 Codex 可信目录下所有根线程的本地 `token_count` 日志，展示输入、缓存输入、输出和总量的实时速率与累计用量。
  - 用量范围支持当前目录或全部目录；首次启动会同步可信目录到可编辑的 JSON 注册表，并保留手动禁用项。
- 新增“精细柱状”曲线风格。
- 新增8种配色方案，将“红蓝”重命名为“冰与火”。

## v2.2.0 - 2026-07-15

- 大幅提升加载和刷新速度
  - 优化历史日志读取与索引方式，避免界面刷新时反复全量解析月度 JSONL。
  - 增加历史数据和主面板缓存，降低高频点击、切换与重绘的响应延迟和内存占用。

## v2.1.0 - 2026-07-15

- 兼容 Codex 临时仅返回 7d 额度窗口的情况，改为根据服务端周期长度识别 5h/7d，不再固定将主窗口视为 5h。
- 历史记录读取时自动纠正此前误存到 5h 键下的 7d 数据，保持额度统计和历史曲线连续，且不改写原始日志。
- “额度窗口”设置现在统一控制看板、专注和合并模式下的当前额度、总额度与历史曲线展示。
- 只有一个有效或已选展示周期时，该周期统一使用更明亮的优先配色。

## v2.0.2 - 2026-07-10

- 新增额度上涨平滑处理，避免服务端瞬时异常值造成看板和历史曲线大幅跳动。

## v2.0.1 - 2026-07-10

- 修复 WSL 重启后残留 PID 被其他进程复用，导致后台额度采样器无法自动恢复的问题。
- 启动和停止脚本现在会校验 PID 对应的真实进程，避免误判或误杀无关进程。
- 采样器改用内核级文件锁，防止重复启动，并新增进程锁回归测试。

## v2.0.0 - 2026-07-06

- 代码重构大更新：入口保留在 `src/codextop.py`，认证、核心逻辑、额度采样和终端界面分别整理到 `auth/`、`core/`、`quota/`、`ui/`。
- 新增自动更新功能。
  - 每天首次打开 CodexTOP 时后台检测 GitHub 更新，发现新版本后可在侧栏通过 `F11` 或点击确认更新。
  - 新增 `CODEXTOP_UPDATE` 命令，支持检测更新和 fast-forward 自动同步，并在更新时展示 Git 命令输出。
- 调整配色方案文件结构：内置样式迁移到 `src/ui/styles/{key}.json`，每个配色方案单独维护。
- 更新安装脚本和文档。
- 新增开源协议。

## v1.2.0 - 2026-07-06

- 新增设置模式，便于小屏幕显示和不支持鼠标的终端操作。
  - 点击当前选项以展开单项选项
  - 新增设置键盘操作：`F9` 进入设置项选择，上下切换设置项，`Enter` 展开全部选项，上下切换选项，`Enter` 确定选项，`Esc` 回退。
- 新增多种曲线风格，更精细的显示效果。
- 支持导入或编写配色方案，新增两种配色方案。
- 新增额度历史曲线窗口筛选，现在可以只展示 5h 或者 7d 的历史曲线，或者同时展示两者。

## v1.1.1 - 2026-07-05

- 修复总额度历史曲线的断档预测逻辑：先按账号独立预测，再汇总为合并额度，避免单个账号重置时被误画成总额度满额。
- 总额度详情行改为完整显示重置文案；账号额度列表过长时仅对账号额度信息使用省略号。
- 新增合并模式示例。

## v1.1.0 - 2026-07-05

- 新增合并模式。
- 合并模式顶部展示账号信息、重置次数和总额度进度条，底部展示合并额度历史。
- 合并统计基于所有账号实时采样汇总，跳过读取失败的账号。

## v1.0.2 - 2026-07-02

- 优化日志数据保存加载机制。
- 历史记录迁移为按月存储。
- 非 `all` 历史读取范围限制为当前月和上个月。
- 兼容旧记录账号 ID 规范化。

## v1.0.1 - 2026-07-02

- 优化界面显示。
- 改进账号展示、历史曲线和紧凑终端布局。
- 更新 README 文档和示例。

## v1.0.0 - 2026-07-02

- 初始发布。
- 提供 CodexTOP 终端看板、单次额度查询、后台采样和多账号切换基础能力。
